# ERP Logistics Frontend Demo

Simple Next.js + Ant Design frontend demo for an internal logistics ERP.

## Setup

```bash
npm install
npm run clean
npm run dev
```

Open `http://localhost:3000`.

## Build Static Output

This frontend is configured to export as a static site. To generate the deployment-ready `out` folder, run:

```bash
npm run clean
npm run build
```

What this does:

- `npm run clean` removes the local `.next` build cache.
- `npm run build` runs `next build`.
- Because `next.config.mjs` uses `output: 'export'`, Next.js writes the static site to the `out` folder.
- After the build finishes, `postbuild` copies `web.config` into `out` for IIS/Plesk hosting.

The generated `out` folder contains:

- Static HTML files for each route
- Next.js assets under `_next/`
- Route data files generated by the App Router export
- `web.config` for rewrite/default document behavior on IIS

You can deploy the contents of `out` directly to a static hosting target or IIS/Plesk environment.

## API

The Axios client uses this base URL by default:

```bash
http://localhost:3003/api/v1
```

To change it, create `.env.local`:

```bash
NEXT_PUBLIC_API_BASE_URL=http://localhost:3003/api/v1
```

Implemented endpoint calls:

- `POST /auth/login`
- `GET /auth/me`
- `GET /jobs`
- `GET /jobs/:id`
- `GET /accounting/revenue/job/:jobId`
- `GET /accounting/cost/job/:jobId`
- `GET /accounting/profit/job/:jobId`
- `GET /partners`
- `GET /accounting/revenue`
- `GET /accounting/cost`

The frontend is mapped to the NestJS backend in `hr-app-localsetup`, where:

- backend port is `3003`
- API prefix is `/api/v1`
- CORS already allows `http://localhost:3000`

If the API is not running, the app falls back to mock data so the UI remains navigable.
